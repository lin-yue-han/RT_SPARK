# RT-SPARK 项目代码总结报告

## 一、软件流程图及说明

### 1.1 系统启动流程

```
系统上电 → RT-Thread 内核初始化 → main() 入口
        ↓
  motor_init()：初始化 PA0/PA1/PB14/PB15 为输出，置低电平
        ↓
  forward()：左/右电机正转，小车前进
        ↓
  finsh 控制台就绪，等待 MSH 命令
```

### 1.2 传感器初始化流程（`sensor_init` 命令）

```
sensor_init 命令
    ↓
bn0055_init()：
  ├─ rt_device_find("uart1")        // 查找 UART1 设备
  ├─ rt_sem_create("bn0055_sem")   // 创建接收信号量
  ├─ rt_device_open(INT_RX)         // 中断接收模式打开 UART
  ├─ 配置波特率 115200
  └─ rt_device_set_rx_indicate()    // 注册接收回调
    ↓
sht3x_init()：
  ├─ rt_i2c_bus_lookup("i2c1")    // 查找 I2C 总线
  ├─ 发送软复位命令 0x30A2
  ├─ 发送测量命令 0x2C06 (单发高精度)
  └─ 读取状态寄存器，CRC-8 校验
    ↓
传感器就绪，可开始数据采集
```

### 1.3 电缆舞动检测流程（`galloping_start` 命令）

```
galloping_start
    ↓
gd_create("cable1", 20, 64)  // 创建检测器：20Hz，64点窗口
    ↓
创建采集线程（优先级12，栈2048字节）
    ↓
[线程循环 50ms 周期]
    ├─ bn0055_read(&bno)        // 读取 BNO055 加速度+欧拉角
    ├─ gd_feed(det, ax,ay,az, roll,pitch)
    │     ├─ 滑动均值滤波（MAF，8点）
    │     ├─ 指数平滑去直流（α=0.001）
    │     ├─ 写入窗口缓冲区
    │     └─ 窗口满？→ gd_analyze()
    ├─ gd_analyze()
    │     ├─ 特征提取（振幅/频率/能量/扭转）
    │     └─ 状态分类（5级判定）
    └─ 状态变化？→ on_state_changed() 打印告警
    ↓
[循环继续，每 3.2s 输出一次分析结果]
```

## 二、算法原理

本系统采用三层数据处理管道，将 BNO055 输出的原始加速度转换为可解释的舞动状态等级。整体处理流程为：**原始采样 → 滑动均值滤波 → 指数平滑去直流 → 多维特征提取 → 规则引擎状态分类**。

### 2.1 滑动均值滤波（Moving Average Filter, MAF）

传感器输出的原始加速度含有高频量化噪声和振动干扰，若直接用于特征提取会严重影响精度。本系统采用长度为 `N=8` 的 MAF 对三轴加速度分别滤波。其基本公式为：

$$y[n] = \frac{1}{N}\sum_{i=0}^{N-1} x[n-i]$$

实现上维护环形队列与运行和 `maf_sum`，每次入队时先减去被覆盖的旧值、再加入新值，均值更新的时间复杂度为 $O(1)$，非常适合嵌入式实时系统。窗口长度 $N=8$ 在 20 Hz 采样率下对应 400 ms 的平均窗口，能有效抑制高于 1.25 Hz 的噪声，同时不损失电缆舞动（0.1～3 Hz）信号的关键信息。

### 2.2 指数平滑去直流

加速度计受重力影响，静止时三轴输出含有约 $9.81\ \mathrm{m/s^2}$ 的直流偏置（重力投影）。若不去除，振幅计算将被严重污染。本系统采用指数平滑估计直流分量：

$$dc_{new} = dc_{old} \cdot (1-\alpha) + x_{filt} \cdot \alpha, \quad \alpha = 0.001$$

$\alpha$ 取 0.001 意味着时间常数约为 $\tau = 1/(\alpha \cdot f_s) = 50\ \mathrm{s}$，足以跟踪传感器缓慢改变的姿态（即重力投影变化），却完全无法跟随 0.1 Hz 以上的振动。动态加速度由滤波后信号减去直流估计得到：

$$a_{dynamic}[n] = x_{filt}[n] - dc[n]$$

### 2.3 特征提取

每当采集窗口（64 点，约 3.2 s）填满后，系统对窗口内的动态加速度序列提取以下五类特征。

**① 振幅（峰-峰值）**：记录窗口内每轴的最大值与最小值之差，反映振动幅度：

$$A_{pp} = \max(a_{dynamic}) - \min(a_{dynamic})$$

主导轴振幅取三轴峰-峰值的最大值。

**② 位移估算**：电缆舞动可近似为正弦振动 $x(t)=A\sin(\omega t)$，其加速度 $a(t)=-A\omega^2\sin(\omega t)$。由此反推位移幅值：

$$A_{disp} \approx \frac{A_{pp}}{2\omega^2} = \frac{A_{pp}}{2(2\pi f)^2}$$

该公式无需积分运算，在嵌入式平台上以浮点乘除即可完成，计算量极小。

**③ 频率估计（过零法）**：统计窗口内信号符号发生变化（即穿越零轴）的次数 $N_{ZC}$，过零率除以 2 即为主频估计：

$$f_{est} = \frac{ZCR}{2}, \quad ZCR = \frac{N_{ZC}}{T_{window}}$$

过零法相比 FFT 无需额外数学库，计算代价为 $O(N)$，在电缆舞动低频段（0.1～5 Hz）精度足够。

**④ 振动能量（RMS）**：加速度信号的均方根值综合反映整体振动强度：

$$E_{RMS} = \sqrt{\frac{1}{N}\sum_{i=1}^{N} a_i^2}$$

**⑤ 扭转特征**：电缆覆冰后截面不对称，舞动时 X、Y 轴振动相位出现差异，表现为两轴过零次数不一致。系统将此归一化为等效扭转角：

$$\theta_{torsion} = \mathrm{clip}\!\left(\frac{|ZC_x - ZC_y|}{0.1N},\ 0,\ 1\right)\times 45°$$

### 2.4 状态分类算法

系统以特征向量为输入，采用基于物理机理的规则引擎进行五级分类，判定优先级从高到低如下：

| 状态 | 中文含义 | 判定条件（优先级从高到低）|
|------|---------|--------------------------|
| `UNKNOWN` | 数据异常 | 置信度 < 0.5（样本不足半窗） |
| `IDLE` | 静止 | 振幅 < 0.03g 且 RMS < 0.03g |
| `ICE` | 覆冰舞动 | $f<0.3\ \mathrm{Hz}$ **且** $A>0.2g$ **且** $\theta>15°$ |
| `SEVERE` | 剧烈舞动 | $f<0.5\ \mathrm{Hz}$ **且** $A>0.4g$ |
| `MODERATE` | 中等舞动 | $f<1.5\ \mathrm{Hz}$ **且** $A>0.15g$ |
| `BREEZE` | 微风振动 | $f>1.5\ \mathrm{Hz}$ **且** $A>0.05g$ |
| `IDLE` | 静止（默认） | 其余情况 |

**覆冰工况的物理依据**：导线覆冰后线质量密度增大，由弦振动理论，固有频率 $f_n = \frac{n}{2L}\sqrt{T/\mu}$ 中线密度 $\mu$ 增大，导致 $f_n$ 显著下降至 0.3 Hz 以下；覆冰截面非对称同时激发扭转振动，三特征（低频 + 大振幅 + 明显扭转）联合判定，可有效区分覆冰舞动与普通大风舞动。

## 三、各模块在系统中的作用

本系统由六个功能模块组成，按"感知→算法→应用→执行"四层架构组织，各模块职责明确、相互解耦。

### 3.1 感知层——BNO055 驱动（`gy_bn0055.c/h`）

GY-BNO055 是系统最核心的感知部件。BNO055 内置九轴 IMU（三轴加速度计 + 三轴陀螺仪 + 三轴磁力计）及片上传感器融合处理器，能直接输出经校准的加速度、欧拉角和四元数，省去外部姿态解算。本模块通过 UART1（PB6/PB7，115200 baud）与传感器通信，采用中断接收模式配合信号量机制，避免轮询占用 CPU。数据帧格式为 NMEA 风格的字符串（`$BNO055,ACC=...,GYR=...,EUL=...*CS`），驱动内部用状态机逐字段解析，支持最多 3 次重试以跳过噪声帧。该模块输出的三轴加速度（单位 m/s²）和欧拉角（°）直接提供给舞动检测算法，是整个数据链路的起点。

### 3.2 感知层——SHT3X 驱动（`sht3x.c/h`）

SHT3X 温湿度传感器通过软件 I2C（SCL→PD10，SDA→PD8）连接，采用单发高精度命令（0x2C06）触发测量，读取 6 字节原始数据并对每组（温度/湿度）进行 CRC-8 校验（多项式 $x^8+x^5+x^4+1$）以确保数据可靠性。温度转换公式为 $T = -45 + 175 \times S_T / 65535$，湿度为 $RH = 100 \times S_{RH} / 65535$。温湿度数据可辅助判断覆冰风险：当温度低于 0 °C 且相对湿度高于 85% 时，覆冰概率显著提升，可与加速度特征联合决策。

### 3.3 算法层——舞动检测算法核心（`galloping_detect.c/h`）

这是整个系统的智能核心，实现完整的三层信号处理管道。对外提供面向对象风格的 API：`gd_create()` 动态分配检测器实例，`gd_feed()` 逐样本喂数据，窗口满时自动触发 `gd_analyze()` 完成特征提取与状态分类，`gd_last_feature()` 供外部查询最新结果。内部维护环形缓冲区、MAF 环形队列和指数平滑直流跟踪器三套数据结构，所有内存均通过 `rt_malloc` 动态分配，支持多实例并行检测不同测量点。该模块完全与硬件解耦，单元可测试性强，是系统可维护性和可扩展性的关键所在。

### 3.4 应用层——舞动检测应用（`galloping_app.c`）

该模块承上启下，将算法层封装为 RT-Thread 应用。它创建一个优先级 12、栈 2048 字节的采集线程，以 50 ms（20 Hz）为周期读取 BNO055 并调用 `gd_feed()`，同时监测状态变化——一旦状态发生跳转，立即调用 `on_state_changed()` 以不同告警级别（INFO / WARN / ALERT / DANGER）向控制台输出提示，包括估算位移、频率和扭转角等关键参数。对外通过四条 MSH 命令（`galloping_start/stop/stat/reset`）提供完整的运行时控制接口，便于调试与现场标定。

### 3.5 执行层——电机控制（`main.c`）

系统入口同时承担电机驱动职责。`motor_init()` 将四路 GPIO（PA0/PA1 控制左电机，PB14/PB15 控制右电机）配置为推挽输出并初始化为低电平（停止状态），`main()` 调用后立即执行 `forward()`，使小车上电即前进，适用于无人巡检场景。五条运动命令（前进/后退/左转/右转/停止）均通过 MSH 导出，支持实时遥控。该模块采用开环方向控制，结构简洁，后续可扩展 PWM 调速或编码器闭环。

### 3.6 传感器应用层（`sensor_app.c`）与 UART4 接收（`uart_recv.c`）

`sensor_app.c` 将传感器初始化封装为 MSH 命令 `sensor_init`，避免系统启动阶段因 UART 初始化阻塞 finsh 控制台（这是调试期间"有 msh 但无法输入"问题的根本解决方案）。`uart_recv.c` 提供独立的 UART4 接收框架，预留给 GPS 模块或上位机通信使用，通过信号量驱动的中断接收线程实现非阻塞数据采集。

## 五、关键设计决策说明

1. **上电即前进**：`main()` 中 `motor_init()` 后直接调用 `forward()`，无需手动干预，适用于自主巡检场景。

2. **传感器初始化延迟**：`sensor_init` 设为 MSH 命令而非自动初始化，避免启动阶段 UART 冲突导致 finsh 无法输入。

3. **MAF + 指数平滑组合滤波**：MAF 抑制高频噪声，指数平滑分离重力，两者结合适应电缆振动信号特点（0.1~5Hz 有用信号，9.8m/s² 重力直流）。

4. **过零法测频**：相比 FFT（计算量大，需额外库），过零法在嵌入式平台上零额外依赖，且满足电缆舞动低频检测需求。

5. **规则引擎而非机器学习**：嵌入式平台资源受限，规则引擎可解释性强、无需训练数据，且电缆舞动物理特征明确，规则判定已足够有效。
