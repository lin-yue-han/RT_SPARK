# RT_SPARK 重启电脑后启动说明

这份文档只讲“电脑重启后怎么把系统重新跑起来”。照着做即可。

## 一、先确认网络

1. 打开手机热点。
2. 电脑连接手机热点。

当前给 ESP32 配置的热点需要和电脑连接的热点一致。请不要把真实热点密码提交到公开仓库，下面用占位符表示：

```text
热点名称：<你的手机热点名称>
热点密码：<你的手机热点密码>
```

ESP32 也会连接这个热点，所以电脑和 ESP32 必须在同一个热点下面。

## 二、确认电脑 IP

在 PowerShell 里运行：

```powershell
Get-NetIPConfiguration | Where-Object { $_.IPv4Address -and $_.NetAdapter.Status -eq 'Up' } | Select-Object InterfaceAlias,@{n='IPv4';e={$_.IPv4Address.IPAddress}}
```

如果看到电脑 Wi-Fi IP 还是：

```text
192.168.43.128
```

那 ESP32 不需要重新配置。

如果 IP 变了，比如变成 `192.168.43.xxx` 的别的数字，就需要重新给 ESP32 写一次服务器 IP。见本文第七部分。

## 三、打开 Sakura Frp 隧道

这一步是给 4G 通信模块 Air778E 用的。

1. 打开 Sakura Frp 启动器。
2. 登录账号。
3. 找到 RT_SPARK 对应的 TCP 隧道。
4. 点击“启动”。
5. 状态显示“运行中”即可。

注意：如果使用 Sakura Frp 启动器，一般不需要手动运行 `frpc.exe`。启动器已经帮你做了隧道转发。

## 四、启动本地桥接服务

打开 PowerShell，运行：

```powershell
cd D:\AI_Intro\RT-ThreadStudio\workspace\RT_SPARK\tools\wireless_bridge
node bridge.js
```

如果系统提示找不到 `node`，用这个命令：

```powershell
cd D:\AI_Intro\RT-ThreadStudio\workspace\RT_SPARK\tools\wireless_bridge
C:\Users\Administrator\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe bridge.js
```

看到类似下面内容，说明桥接服务正常：

```text
TCP Server 监听 0.0.0.0:8090
WebSocket Server 监听 ws://localhost:8080
```

这个窗口不要关。关了以后网页就收不到无线数据。

## 五、打开网页

浏览器打开：

```text
file:///D:/AI_Intro/RT-ThreadStudio/workspace/RT_SPARK/tools/wireless_bridge/cable-monitor.html
```

或者打开：

```text
http://localhost:8080/
```

如果网页左上角显示 WebSocket 正常，说明网页和电脑桥接服务已经连上。

## 六、给设备上电

推荐顺序：

1. 先打开手机热点。
2. 电脑连上手机热点。
3. 启动 Sakura Frp 隧道。
4. 启动 `bridge.js`。
5. 打开网页。
6. 给星火 1 号、4G 模块、电机供电。
7. 给 ESP32 + BNO055 单独供电。

ESP32 不需要连接电脑。它会自动连接你配置过的手机热点，然后把 BNO055 加速度数据发到电脑 `192.168.43.128:8090`。

## 七、如果电脑 IP 变了，重新配置 ESP32

ESP32 现在记住的服务器 IP 是：

```text
192.168.43.128
```

如果电脑重启后热点分配了新的 IP，需要把新 IP 写入 ESP32。

先把 ESP32 用 USB 数据线接到电脑，然后运行：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File D:\AI_Intro\RT-ThreadStudio\workspace\RT_SPARK\tools\esp32_bno055\configure_esp32_wifi.ps1 -Port COM6 -Ssid "<你的手机热点名称>" -Password "<你的手机热点密码>" -BridgeHost "你的新电脑IP" -BridgePort 8090
```

例如如果新 IP 是 `192.168.43.55`，就运行：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File D:\AI_Intro\RT-ThreadStudio\workspace\RT_SPARK\tools\esp32_bno055\configure_esp32_wifi.ps1 -Port COM6 -Ssid "<你的手机热点名称>" -Password "<你的手机热点密码>" -BridgeHost "192.168.43.55" -BridgePort 8090
```

写入完成后，ESP32 可以拔掉 USB，改为单独供电。

## 八、检查系统是否正常

打开 PowerShell 运行：

```powershell
Invoke-RestMethod http://localhost:8080/health | ConvertTo-Json -Depth 8
```

重点看：

```text
tcpConnections
wsConnections
totalMessages
```

含义：

```text
wsConnections > 0：网页已经连上 bridge.js
tcpConnections >= 1：至少有一个无线设备接入，例如 Air778E 或 ESP32
totalMessages 一直增加：正在收到数据
```

## 九、常见问题

### 1. 网页显示 WebSocket 断开

说明 `bridge.js` 没启动，或者 8080 端口被占用。

解决：

```powershell
cd D:\AI_Intro\RT-ThreadStudio\workspace\RT_SPARK\tools\wireless_bridge
node bridge.js
```

### 2. 网页显示 WebSocket 正常，但没有 ESP32 加速度数据

按顺序检查：

1. 手机热点是否打开。
2. 电脑是否连接这个热点。
3. ESP32 是否上电。
4. 电脑 IP 是否还是 `192.168.43.128`。
5. 如果电脑 IP 变了，按第七部分重新配置 ESP32。

### 3. Air778E 显示接入，但没有 ESP32 数据

这只能说明 4G 通信模块连上了，不代表 ESP32 连上了。

ESP32 正常时，网页会收到：

```json
{"type":"raw_accel","source":"esp32_bno055","ax":0.01,"ay":0.00,"az":-0.02}
```

### 4. 4G 模块没有数据

按顺序检查：

1. Sakura Frp 隧道是否运行中。
2. `bridge.js` 是否运行。
3. 4G 模块是否亮灯、SIM 卡是否正常。
4. 星火 1 号和 4G 模块供电是否稳定。

### 5. 电机控制没反应

按顺序检查：

1. 网页 WebSocket 是否正常。
2. Sakura Frp 隧道是否运行。
3. Air778E 是否接入。
4. 星火 1 号和 4G 模块是否一起稳定供电。
5. 电机电源是否接好。

## 十、一句话版启动流程

```text
开手机热点 → 电脑连热点 → 启动 Sakura Frp 隧道 → 运行 bridge.js → 打开网页 → 给星火1号/4G/ESP32 上电
```
